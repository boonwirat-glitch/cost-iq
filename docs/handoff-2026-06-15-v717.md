## Echo v2 Architecture — สิ่งที่ทำใน session นี้

### Goal
Build Echo ให้ผ่าน criteria ใน spec:
- Thai accuracy > 90%
- Hallucination < 5%
- Speaker attribution > 90%
- Evidence coverage 100% (ทุก fact มี segment_id)
- Unsupported fact rate = 0%

### 1. Worker v4 (`worker/freshket-sense-ai-proxy.js`)

**เปลี่ยนจาก v3:**

| ส่วน | เดิม | ใหม่ |
|------|------|------|
| `/transcript` output | `{ts, speaker, text}` | `{segment_id, ts, start_sec, end_sec, speaker, text, speaker_confidence, transcript_confidence}` |
| Diarize fail | throw "ไม่มี segments" | fallback → Whisper raw segments, speaker="ไม่ทราบ" |
| Timeout | fixed 6 min | dynamic: max(6min, recording_secs × 1.5) |
| `/analyze` retry | ไม่มี | 3 attempts รวม JSON parse fail |
| `/eval` endpoint | ไม่มี | ✓ วัด Thai accuracy, speaker accuracy, hallucination, evidence coverage |

**Fallback chain (spec principle: transcript never lost):**
```
Groq Whisper → Gemini diarize → ถ้า segments=[] → ใช้ Whisper raw, speaker="ไม่ทราบ"
```

### 2. `09_conv_intel.js` — `_processBlob` rewrite

**เปลี่ยนจาก v709:**

| ส่วน | เดิม | ใหม่ |
|------|------|------|
| Save order | save ทุกอย่างท้ายสุด | save transcript ก่อนเสมอ (step 1) |
| Summary fail | silent `{}` | explicit `null` → Claude รู้ว่าไม่มี summary |
| Transcript timeout | fixed 6 min | dynamic ตาม `_secs` |
| Error message | generic | แยกระหว่าง "transcript saved" vs "ไม่ได้ save" |

**New save architecture:**
```
_saveTranscriptOnly()        → insert ci_sessions (stage='transcribed') ทันทีหลัง step 1
_saveAnalysisToExistingSession() → update row เดิมด้วย skills+OCPB (stage='analyzed')
```

### 3. DB Migration (`sql/echo_v2_migration.sql`)

Run ใน Supabase SQL editor:
- `ci_sessions.transcript_source` column ใหม่
- `echo_eval_log` table — เก็บ eval scores ต่อ session
- `echo_eval_summary` view — monitoring dashboard
