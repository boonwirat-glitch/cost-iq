### 1. Transfer Out Adjusted Base NRR (v776)

**Logic เปลี่ยน:**
- transfer_out outlet ที่อยู่ใน core cohort (baseMap) จะถูกหักออกจาก `base_norm` แบบ retroactive ทั้ง Q
- outlet ที่มาจาก transfer_in / handover / new_sales / comeback ที่ย้ายออกในภายหลัง → ไม่หักฐาน (ไม่อยู่ใน baseMap)
- NRR ทุกเดือน (เม.ย./พ.ค./มิ.ย.) คำนวณจาก `adjusted_base_norm`

**Return object ใหม่จาก `_qnrrCompute`:**
```js
{
  base_norm_original:     number,  // ฐานก่อนหัก
  transfer_out_base_norm: number,  // จำนวนที่ถูกหัก (unit: GMV/day)
  transfer_out_outlets:   Array,   // [{outlet_id, gmv_norm, account_name, period_month}]
  base_norm:              number,  // ฐานที่ปรับแล้ว
}
```

**Bug ที่เจอและแก้ระหว่าง session (v776b):**
- `gmv_norm` ถูก ×30 ผิด unit → ฐานติดลบ → Mar bar แดงทั้งหมด
- แก้: `gmv_norm = b.gmv / b.days` (ไม่ ×30) ให้ตรง unit กับ `base_norm`
