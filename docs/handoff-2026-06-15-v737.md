## สิ่งที่ทำใน session นี้

### Sales Boot White Screen Fix — v737

**Symptom:** Sales user เข้า app หลัง login → จอขาว/ดำ ต้อง hard refresh ถึงจะเห็นข้อมูล

**Root cause ที่แท้จริง (confirmed จาก code + console log):**

Timing race ระหว่าง `body.sales-mode` class กับ `showSenseSplash()`:

```
Sequence:
1. hideLoginOverlay() → showSenseSplash() เริ่มทำงาน
2. showSenseSplash() L602: เช็ค body.classList.contains('sales-mode') → FALSE
   เพราะ sales-mode ถูก set ใน _splashPreFade() ซึ่งรันตอน doFade() เท่านั้น
3. handover ไม่ถูก pre-mark → DataRegistry Tier1 รอ handover
4. FOREGROUND fetch: handover 404 (Sales ไม่มีไฟล์นี้)
   └─ แม้ loadFromCloudflareR2() L1739 จะ pre-mark handover
      แต่ function นี้ถูกเรียกหลัง showSenseSplash() เริ่มแล้ว (race)
5. allCriticalReady() = false ตลอด splash
6. Splash timeout 5000ms → doFade()
   └─ L685: allCriticalReady() still false → _pendingRefreshAll ไม่ถูก clear → ไม่ render
7. จอขาว — ต้อง hard refresh
```

**หมายเหตุ:** v732-v736 พยายามแก้จากหลายจุด แต่ทุกจุดนั้นสมมติว่า body class ถูก set ก่อนแล้ว ซึ่งไม่จริง

**Fixes ใน v737:**

**Fix 1 — showSenseSplash() (01_core.js L600):**
เพิ่ม `getCurrentRole()` fallback นอกเหนือจาก body class check:
```javascript
// v736 (buggy): body.sales-mode ยังไม่ถูก set → _splashRole = false
var _splashRole = body.classList.contains('sales-mode');

// v737 (fixed): getCurrentRole() ใช้ได้เลย — profile โหลดก่อน showSenseSplash
var _splashRole = body.classList.contains('sales-mode');
if (!_splashRole && typeof getCurrentRole === 'function') {
  var _sr = getCurrentRole();
  _splashRole = (_sr === 'sales' || _sr === 'sales_tl');
}
// → handover pre-marked ทันที → allCriticalReady() true → splash fade ปกติ
```

**Fix 2 — doFade() MAX_SHOW safety (01_core.js ~L715):**
ถ้า splash timeout แล้ว Sales ยังไม่ render (fallback case):
```javascript
// v737: หลัง teamview MAX_SHOW path
if (_spLen > 0 && _isSalesNow) {
  DataRegistry.markLoaded('handover');  // force mark
  RenderBus.flushNow();                 // force render
}
```
