## ปัญหาที่อาจพบต่อจากนี้ ❗

### A. R2 filename ไม่ตรงกับ tab key ใหม่
`sense_sku_outlet_{key}.csv` (ชื่อไฟล์ R2) ยังเหมือนเดิม — แค่ IDB tab key ที่เปลี่ยน (`v2-`)
Fix B ไม่กระทบ R2 filename เลย — ถูกต้อง

### B. approaching + outlet tooltip
`_hasOutlet` guard ใน `05_kam_view.js` L742:
```js
const _hasOutlet = _outletData && _outletData.length >= 1 &&
  (s.type === 'gone' || s.type === 'near' || s.type === 'approaching')
```
`approaching` อยู่ในเงื่อนไขแล้ว ✓ — ถ้า outlet data โหลดแล้วและ SKU classify เป็น approaching → tooltip ขึ้น

### C. Portview `approaching` badge ไม่นับใน _churnCounts
`computeChurnCountsForAccount()` (06 L169-198) นับแค่ `gone` และ `near` เท่านั้น — `approaching` และ `not_yet` ไม่ถูกนับ
ดังนั้น portview badge จะไม่แสดง approaching count — design เป็นแบบนั้นตั้งใจ (approaching = ยังดีอยู่ ไม่ urgent)
