## เป้าหมาย session นี้
แก้ rendering inconsistency ระดับโครงสร้าง (ไม่ใช่ point patch) จาก feature/fetch logic ที่ build ทับกันมาหลายเวอร์ชัน — เน้นความเสถียรของการเล่นโฟลวต่างๆ บน PWA

## ปัญหาที่ผู้ใช้รายงาน (5 ข้อ) → root cause
1. PWA admin เข้า account view ไม่เห็น pace bar ทั้งก้อน (web เห็นปกติ)
2. Admin เข้า portview ของ KAM ชื่อยาว → ชื่อขึ้น 2 บรรทัด (อยากให้ trunc)
3. Mobile admin เข้า accountview → SKU movement ช้ามาก (เป็นนาที)
4. portview scroll พับสุดแล้ว scroll กลับ/หลัง search/filter → ไม่กางเต็ม
5. เข้า Echo แล้วกลับมาพอร์ต → search แล้วจอกระตุกเหมือน crash

## Root cause ที่เจอ (เชิงโครงสร้าง)
- **R1 — Observer lifecycle หาย:** `_pvCollapseObserver` (scroll-collapse rAF loop) ไม่ถูก disconnect ตอนออกจาก portview → ค้างข้ามหน้าจอ (Echo/account/teamview) → re-init ทับ → 2+ observers วิ่งพร้อมกัน → rAF storm → กระตุก
- **R2 — Admin ≠ KAM email:** bundle fetch ใช้ kamEmail เป็น key. admin resolve ไม่ได้ตอน portviewBulkData ว่าง → ไม่มี fallback ที่ถูก → ไม่โหลด/รอ 90s timeout
- **R3 — scroll-clamp มองข้าม:** filter ทำ list หด → browser clamp scrollY=0 → `_frame()` ตีความ "y=0 + เคยพับ = ยังพับ" → ค้างพับถาวร
- **R4 — pace bar อยู่ใน pv-collapsible:** pace bar + commission strip อยู่ใน collapsible เดียวกัน → พับค้าง = หายไปด้วย (นี่คือต้นเหตุข้อ 1 ที่แท้จริง — ไม่ใช่ตัว target bar เสีย)

## การแก้ (commit fa20e0e)

### Fix E — generation token กัน observer ซ้อน
`src/06_portview_teamview.js`
- เพิ่ม `let _pvObsGen=0` module-level
- ทุก `_pvInitCollapseObserver()` claim `_myGen=++_pvObsGen`
- guard ทุก `_frame`/`_onScroll`/init-setTimeout/`_pvResetExpandedH`: `if(_myGen!==_pvObsGen)return`
- frame/scroll ของ generation เก่ากลายเป็น no-op → ไม่มี double rAF loop

### Fix E2 + A — teardown ตอน nav-away + fresh entry expanded
`src/05_kam_view.js` (showScreen) + `src/06_portview_teamview.js`
- expose `window._pvTeardownCollapseObserver()` (เลี่ยง TDZ ของ let ข้าม module)
- showScreen: ออกจาก portview → teardown observer + เคลียร์ `_pvLastCollapseMs`/`_pvWasCollapsed`
- ผล: เข้า portview ใหม่ทุกครั้ง (PWA cold/Echo return/teamview drill) เริ่ม expanded → pace bar + commission strip มองเห็น

### Fix E3 — hardened body scroll restore
`src/05_kam_view.js`
- restore `body.overflow` เมื่อ Echo sheet ไม่ active (`.ci-open`) ไม่ใช่แค่เมื่อ sheet หาย → กันเคส PWA ถูก kill กลาง Echo แล้ว scroll lock ค้าง

### Fix D — not-scrollable guard
`src/06_portview_teamview.js` `_frame()`
- ย้าย not-scrollable guard ขึ้นมาก่อนสุด + unconditional
- ถ้า page scroll ไม่ได้ → กางกลับเสมอ + เคลียร์ snapshot (ไม่ติดกับพับ)
- supersede few-cards guard เดิม (ที่ skip เมื่อ wasCollapsed) และ y===0 collapse-restore

### Fix D2 — snapshot parity
- `setPvSort()`/`setPvView()` snapshot `_pvWasCollapsed` เหมือน `schedulePortviewListRender()`

### Fix C — admin/TL bundle retry
`src/06_portview_teamview.js` `portviewSelectAccount()`
- admin/TL ที่ resolve kamEmail ไม่ได้ (portviewBulkData ว่าง) → **ไม่** fetch ด้วย admin email (ต้นเหตุ 90s timeout)
- poll resolve kamEmail สูงสุด ~3.2s (8×400ms) แล้วค่อย fetch bundle ที่ถูก
- หยุด retry ถ้า user navigate ออก

### Fix B — KAM name truncation
`src/styles_portview.css`
- `#portview-heading`: overflow:hidden + text-overflow:ellipsis + white-space:nowrap
- `.portview-title`: min-width:0 + flex:1 1 auto; `#pv-action-btns`: flex-shrink:0
