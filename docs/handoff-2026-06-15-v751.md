### Root Cause จริง (พบจาก diagnostic + stack trace)

**`showScreen()` ใน `__legacyShowScreenFallback` ใช้ `querySelectorAll('.scr')` โดยไม่ scope**

```javascript
// BEFORE (bug)
document.querySelectorAll('.scr').forEach(s => s.classList.remove('on'));

// AFTER (fix)
document.querySelectorAll('.scr:not(#ci-fullsheet .scr)').forEach(s => s.classList.remove('on'));
```

`querySelectorAll('.scr')` เลือกทุก element ที่มี class `scr` ใน document รวมถึง `ci-s-record`, `ci-s-proc`, `ci-s-result` ใน `#ci-fullsheet` (Echo sheet)

**Timeline ที่เกิด:**
1. User กด Echo → `CI.open()` → `_mount()` → append sheet → `_showScreen('ci-s-record')` → add `on` ✓
2. Supabase auth `_recoverAndRefresh` → `_notifyAllSubscribers` → `hideLoginOverlay()` → splash fallback timer fire
3. `_autoRouteAfterLogin()` → `showScreen('sales-portview')` → `querySelectorAll('.scr').remove('on')` ← **ลบ `on` จาก Echo screen ด้วย**
4. Echo sheet ยังอยู่ใน DOM แต่ไม่มี screen active → จอขาว

**เหตุที่หาไม่เจอนาน:** Heisenbug — การรัน script ใน console เพิ่ม JS execution delay เล็กน้อย ทำให้ race window แคบเกินไปจน bug หายไป

## Key Learnings จาก session นี้

1. **`querySelectorAll` ไม่ scope โดย default** — ทุก global query ต้องคิดว่า "มี dynamic element อื่นใน DOM ไหมที่ match selector เดียวกัน" โดยเฉพาะ `#ci-fullsheet` ที่ mount/unmount บน body
2. **Heisenbug = race condition แคบมาก** — ถ้า bug หายเมื่อรัน script ใน console แปลว่า timing เป็นตัวการ ไม่ใช่ logic ผิด
3. **Stack trace คือ clue ที่ดีที่สุด** — `[login transition] splash fallback release` ใน stack บอกทันทีว่า `showScreen` ถูก call จาก auth event
4. **อย่าแก้ที่ symptom** — จอขาวเพราะ screen ไม่มี `on` แต่สาเหตุไม่ได้อยู่ที่ Echo code เลย อยู่ที่ `showScreen` ใน shell
