## สิ่งที่ทำใน session นี้

### 1. Login Stability (v714→v715)

**ปัญหาที่แก้:** หลัง login เลขกระพริบ 3-4 รอบ, teamview/portview โหลดไม่ครบแล้ว refresh, Echo กด แล้วจอขาว, Sales portview แสดง 0 ก่อนข้อมูลมา

**Root cause:** RenderBus._flush() ถูกเรียกผ่าน flushNow() โดยตรง ข้าม Tier 1 guard ใน signal() → render ก่อนข้อมูลครบ

**สิ่งที่แก้ (v715):**

| Fix | File | สิ่งที่เปลี่ยน |
|-----|------|----------------|
| 1B — RenderBus Tier guard | `src/02_data_pipeline.js` | เพิ่ม Tier 1 guard ใน `_flush()` โดยตรง — ถ้า Tier 1 ยังไม่ ready → hold + re-queue |
| 2A — skeleton utility | `src/01_core.js` | เพิ่ม `window._showScreenSkeleton(id)` / `window._hideScreenSkeleton(id)` |
| 2B — skeleton clear | `src/02_data_pipeline.js` | `_flush()` call `_hideScreenSkeleton()` ก่อน render ทุก screen |
| 2C — Sales skeleton | `src/10_sales_view.js` | `renderSalesPortview()` แสดง shimmer ถ้า Tier 1 ยังไม่พร้อม |
| 3A — Echo mount-first | `src/09_conv_intel.js` | `echoOpen()` เรียก `CI.open()` ทันทีเสมอ ไม่ block silent อีกต่อไป |

**หมายเหตุ:** v714 ทำก่อนหน้าแล้ว revert เพราะ Fix 1A (เปลี่ยน routing ใน `_autoRouteAfterLogin`) ทำให้ warm boot ไม่ route → จอขาว v715 ไม่แตะ routing เลย แก้เฉพาะ RenderBus และ skeleton
