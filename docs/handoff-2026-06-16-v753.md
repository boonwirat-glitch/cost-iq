### Fix A: Restaurant sheet white bg (`src/styles_restaurant.css`)

**Root cause:**  
v752 เพิ่ม `body.kam-mode #rest-sheet-body{background:var(--tk-bg-base)}` เพื่อแก้ account view จอขาว  
แต่ rule นี้ apply ตลอดเวลาที่อยู่ใน KAM mode — รวมถึงตอน restaurant sheet กาง  
ทำให้ sheet พื้นหลัง dark แทนที่จะเป็นขาว

**Fix:**  
เพิ่ม rule ที่ specific กว่าโดยใช้ `body.restaurant-sheet` (active เฉพาะตอน sheet กาง):
```css
body.restaurant-sheet #rest-sheet-body{background:#ffffff}
body.restaurant-sheet #swipe-grp-a .scr{background:#ffffff!important}
body.restaurant-sheet #swipe-grp-b .scr{background:#ffffff!important}
```
Account view (body.kam-mode ไม่มี restaurant-sheet) ยังคง dark bg ตามเดิม

### Fix B: Admin bundle prewarm ครอบคลุมทุก KAM (`src/02_data_pipeline.js`)

**Root cause:**  
`_senseGetTlPrewarmEmails(8)` — prewarm แค่ top 8 KAM by GMV  
`_startTlBundlePrewarm` — fetch sequential ทีละคน (await + 200ms sleep)  
Admin เห็นทุก KAM → account ของ KAM นอก top 8 = badge ว่างจนกว่าจะ click เข้าไป

**Fix:**  
- `maxCount` default: 5 → 999 (ดึงทุก KAM ในทีม)
- เปลี่ยนจาก sequential → parallel batch ละ 4 (`Promise.all`)
- render portview หลังแต่ละ batch → user เห็น badge ทยอยขึ้น

**Root cause:**

**Fix:**

---

### Fix D: SKU badge cache invalidation (`src/02_data_pipeline.js`)

**Root cause:**  
เมื่อ `sku_current.csv` โหลดเสร็จ → portview account cache (`_pvAcctCacheResult`) ไม่ถูก clear  
→ `computeChurnCountsForAccount()` คืน null (จาก cache เก่า) → badge ว่าง  
`DataRegistry.markLoaded('sku_current')` ไม่ถูกเรียกนอก currentAccountId block → version ไม่ bump → RenderBus skip render

**Fix:**  
- `DataRegistry.markLoaded('sku_current')` เรียกเสมอ (ไม่ gate ด้วย currentAccountId)
- `_pvClearAccountCache()` เรียกเสมอเมื่อ sku_current โหลด
- `RenderBus.signal('sku_current')` เรียกเสมอ (ไม่ gate)
