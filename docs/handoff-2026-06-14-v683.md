## สิ่งที่ทำใน session นี้ (v673–v683)

### Echo topbar safe-area fix (v673–v683)
**Root cause ที่แท้จริง:** `styles_layout.css` มี global rule `.topbar { position:fixed; top:0; padding-top:calc(11px + env(safe-area-inset-top)) }` — Echo sheet ใช้ class `.topbar` เดียวกัน จึงโดน `position:fixed` leak เข้ามา ทำให้ topbar ลอยออกไปทับ status bar

**Fix (v679):** `#ci-fullsheet .topbar` override ด้วย `position:relative; top:auto; left:auto; right:auto; max-width:none; z-index:1; margin:0; padding:16px 24px 10px` — reset ทุก property ที่ global rule inject มา

**Fix (v680):** ลบ padding ออกจาก `#ci-fullsheet` container เหลือแค่ `padding-top:env(safe-area-inset-top,44px)` เพื่อ guarantee safe-area บน iOS ทุก device (44px fallback สำคัญ — อย่าลบ)

### Echo UX fixes (v681)
- **ECHO label ไม่อยู่กลาง:** `.tb-lbl` เพิ่ม `position:absolute; left:50%; transform:translateX(-50%); pointer-events:none` — อยู่กลางเสมอไม่ว่า left/right content จะยาวแค่ไหน
- **Tab bar ตอน recording:** ซ่อน `ci-main-tabs` wrapper ด้วย `parentElement.style.display='none'` ตอน `is-rec` — restore กลับตอน light mode

### Sales mode topbar fix (v673)
**Root cause:** `#scr-sales-portview` ฯลฯ อยู่ใน `swipe-grp-b` ไม่ได้อยู่ใน `.main{padding-top:var(--topbar-h)}` → content โดยไม่มี topbar offset
**Fix:** เพิ่ม `padding-top:var(--topbar-h)` ใน `styles_sales.css` สำหรับ Sales screens ทั้งหมด

### KAM rendering stability (v673–v674)
- **pace=0% flash:** guard ใน `renderPortviewTargetBar()` — return early ถ้า `portviewBulkData.length===0`
- **Stale UI flash on logout:** `_showLoginOverlayClean()` clear innerHTML ของ portview/commission UI elements ทุก logout
- **Collapse strip stale:** force `visible` class เมื่อ restoring collapsed state ที่ `y===0`

### Nav pill 10-icon flash fix (v682)
**Root cause:** HTML มี 14 nav buttons ทั้งหมด visible โดยไม่มี CSS default-hide — NavConfig.render() ถูกเรียกด้วย `setTimeout(0)` หลัง nav กลายเป็น visible แล้ว

**Fix 2 จุด:**
1. **`_splashPreFade`:** เพิ่ม `NavConfig.render()` ก่อน `splash.opacity='0'` — nav ถูกก่อน splash fade
2. **skip-splash path:** เพิ่ม `NavConfig.render()` ก่อน `splash.display='none'` — nav ถูกก่อน visible บน warm boot
3. **Logout nav reset:** `resetRuntimeSessionState()` set nav buttons ทั้งหมด `display:none` — clean slate ทุก session

### Shimmer coverage extend (v682–v683)
- `_activatePortviewShimmer()` ตอนนี้ clear `tgt-portview-bar`, `tgt-nrr-bar`, `pv-commission-strip`, `portview-pace-bar`, `pv-compact-strip` ด้วย
- **SKU badge shimmer (v683):** CSS rule ซ่อน `.portview-acct-badge-row` ระหว่าง shimmer active (`#portview-list:has(#pv-shimmer-wrap)`) — กัน empty→populated flash เมื่อ KAM bundle มาทีหลัง
- **Two-pass `_pvSyncListOffset` (v683):** 0ms + 100ms — กัน cards ทับ header เมื่อ layout reflow ช้า

## Key architecture notes (สำคัญสำหรับ session ต่อไป)

### Echo CSS architecture
```
#ci-fullsheet { padding-top:env(safe-area-inset-top,44px) } ← sheet pad safe-area (44px fallback อย่าลบ)
#ci-fullsheet .topbar { position:relative; top:auto; left:auto; right:auto;
                         max-width:none; z-index:1; margin:0; padding:16px 24px 10px }
← override ทุก property จาก global .topbar{position:fixed}
```

### Nav flash fix architecture
```
_splashPreFade() → set body classes → NavConfig.render() → splash.opacity='0'
skip-splash path → set body classes → NavConfig.render() → splash.display='none'
```
NavConfig.render() ต้องอยู่ **ก่อน** nav กลายเป็น visible เสมอ

### Shimmer lifecycle
```
_activatePortviewShimmer() → pv-shimmer-wrap เกิด → CSS :has() ซ่อน badge rows
_deactivatePortviewShimmer() → pv-shimmer-wrap ถูกลบ → badge rows visible อีกครั้ง
```

### _pvSyncListOffset — two-pass pattern
```js
setTimeout(_pvSyncListOffset, 0);   // frame แรก
setTimeout(_pvSyncListOffset, 100); // fallback หลัง layout settle
```
