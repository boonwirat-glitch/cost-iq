### 1. Filter chip — wrap 2 rows แทน horizontal scroll

**ปัญหา:** chips 7 ตัว (ทั้งหมด/Core/Churn/Handover/Expansion/New/Comeback) scroll แนวนอน ใช้ลำบากบน mobile

**Solution:**
- CSS: `.qnrr-list-filter-bar` เปลี่ยนจาก `flex-wrap:nowrap + overflow-x:auto` → `flex-wrap:wrap`
- chips wrap เป็น 2 rows อัตโนมัติเมื่อล้น
- Dynamic count badges: JS เพิ่มตัวเลขท้ายชื่อ chip (`Core 34`, `Churn 12` ฯลฯ)
- Chips ที่ count=0 ถูก `display:none` อัตโนมัติ

### 2. Sticky month header นอก account table

**ปัญหา:** header เดือน (มี.ค./เม.ย./พ.ค./มิ.ย.) จมอยู่ใน `<thead>` ของแต่ละ account table → ซ่อนเมื่อ account collapse

**Solution:**
- JS: `_qnrrRenderList` เพิ่ม `<div class="qnrr-list-mo-hdr">` เป็น global fixed header เหนือ account list ทั้งหมด
- CSS: `.qnrr-list-mo-hdr` — `display:grid; grid-template-columns:1fr repeat(4,42px)` align กับ table colgroup
- `position:sticky; top:0; z-index:5` — header ติดอยู่ตลอดขณะ scroll
- JS: ลบ `<thead>` ออกจาก individual account tables (ไม่ต้องซ้ำ)

### 3. Partial bar top label — 2 rows ไม่ wrap

**ปัญหา:** Jun partial bar แสดง `฿8.0M actual` และ `17/30d` บรรทัดเดียวกัน → ข้อความถูกบีบ

**Solution:**
- JS: เพิ่ม class `partial` ให้ `qnrr-bar-top-wrap` เมื่อ `bm.is_partial === true`
- CSS: `.qnrr-bar-top-wrap.partial { height: 44px; }` (เพิ่มจาก 32px)
- ทั้ง 2 divs มีพื้นที่เพียงพอ ไม่ต้องเปลี่ยน column width

### 4. Breakdown table — Handover + New Sales cohort view

**ปัญหา:** handover กับ new_sales คือสิ่งเดียวกัน (accounts ที่ถูก hand มาจาก Sales) ต่างแค่ new_user_exp_date month

**Business logic:**
- `handover` = cohort ที่ start Mar (รับ + มี GMV ใน Mar)
- `new_sales` = cohort ที่ start Apr/May/Jun (รับใน Q แต่ไม่มี Mar GMV)

**Solution:**
- JS: BK_ORDER เอา `new_sales` ออก (handled inside handover block)
- JS: Handover row แสดง `Handover & New` = total(handover + new_sales) ทุกเดือน
- JS: Sub-rows ใต้ `└ cohort Mar` = handover GMV, `└ cohort Apr+` = new_sales GMV
- LEGEND_ORDER ยังคง legend แค่ handover (new_sales ไม่แสดงใน legend แยก)
