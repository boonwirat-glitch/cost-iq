**Trigger:** Tap hero card (`#tgt-portview-bar`) ใน portview

- `src/shell.html` — `#qnrr-overlay` + `#qnrr-sheet` HTML

#### Architecture
- **Single CSV** `sense_qnrr_2026q2.csv` (~8,568 rows, 15 KAMs)
- **Index:** `byKamEmail{}`, `byTlEmail{}`, `allRows[]` build ตอน ingest
- **Cold path:** skeleton → fetch → render
- **Warm path (same session):** render ทันที
- **Warm path (IDB):** `'qnrr'` อยู่ใน `_preloadFromIndexedDB()` EXTRA list
- **Background prefetch:** `_prefetchQnrrIfNeeded()` หลัง FOREGROUND done +2s
- **Sales role guard:** ไม่ fetch, ไม่แสดง

#### Compute — `_qnrrCompute(email, scope)` ใน 07c
- **Bug ที่แก้:** base_gmv ถูกนับซ้ำ 3× (outlet ปรากฏ Apr/May/Jun แต่ละ row มี base_gmv)
- **Fix:** `baseMap` (outlet_id → {gmv, days}) build ครั้งเดียว ก่อน loop เดือน
- **NRR%** = `SUM(curr_gmv/curr_days) / SUM(base_gmv/base_days) × 100` (unique outlets only)
- **ตัวเลขที่ถูกต้อง (full portfolio):** Apr≈91%, May≈86%, Jun≈87%

#### SQL — quarterly_nrr_2026_Q2.sql
- Base = Mar 2026, Q months = Apr/May/Jun
- **LEG A:** outlets ที่มี order เดือนนั้น → classify movement
- **LEG B:** transfer_out (Mar cohort + ownership เปลี่ยน)
- **LEG C (สำคัญ):** silent outlets — Mar cohort ไม่มี order เดือนนั้น → churn or transfer_out
  - ใช้ `current_kam_snapshot` จาก `dim.user_master` แยก churn vs transfer_out
- **Expansion:** `first_dollar_date >= 2026-04-01` (align กับ commission SQL)
- **Comeback:** `first_dollar_date < 2026-04-01` + ไม่อยู่ Mar cohort + กลับมา
- **jun_days:** Dynamic `DATE_DIFF(CURRENT_DATE-1, 2026-06-01)+1` (day-1 lag)
- **ต้อง rerun SQL แล้ว export ใหม่** ก่อน upload R2

### 3. Detail Panel Clip — ยังไม่แก้ได้ (OPEN BUG)

**Symptom:** กด "i" บน hero card → detail panel เปิดแต่ถูก crop

**Root cause (confirmed via console):**
```
CLIPS pv-collapsible ov:hidden/hidden maxH:400px elBottom:509 panelBottom:820
pv-collapsible inline maxH: (empty — ไม่มี inline style)
scrollH: 0
```

`400px` มาจาก `_frame()` ที่ set `collapsible.style.maxHeight = expandedH + 'px'`  
`expandedH` ถูก measure ตอน hero card ยังไม่มี detail panel = ~400px  
`_frame()` วิ่งทุก rAF + override maxHeight กลับตลอด

**Approaches ที่ลองแล้ว (ไม่สำเร็จ):**
1. re-measure ใน `_tgtToggleDetail` โดยตรง → `_frame()` override กลับ
2. dirty flag `_pvExpandedHDirty` + scroll dispatch → `_frame()` clear flag แต่ logic ยังไม่ re-measure ถูก
3. expose `window._pvResetExpandedH()` จาก closure → set `expandedH=0` + schedule rAF → panel ยังโดน clip

**สิ่งที่ยังไม่แน่ใจ:**
- `scrollH:0` ตอน query — อาจเกิดจาก `_frame()` วิ่งแล้ว clear `maxHeight=''` ชั่วคราวแล้ว panel collapse กลับก่อน measure
- อาจต้องแก้ที่ portview collapsible logic ให้รู้จัก "expanded due to child content" แทน scroll-based height

**แนวทางที่น่าลอง (ยังไม่ได้ทำ):**
- ใน `_frame()`: ถ้า `collapsible` มี `.tgt-detail-panel.open` อยู่ → `expandedH = max(expandedH, collapsible.scrollHeight)` เพื่อไม่ให้ clip
- หรือเพิ่ม `min-height` แบบ dynamic ให้ collapsible ตาม panel height
