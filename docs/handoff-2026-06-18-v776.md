# Handoff: v776 — Transfer Out Adjusted Base NRR
Date: 2026-06-18

## Summary
เปลี่ยน transfer_out logic ใน QNRR ให้ยุติธรรมต่อ KAM:
outlet ที่ TL ตัดสินใจ transfer ออกไป (และอยู่ใน core cohort)
จะถูกหักออกจากฐาน NRR แบบ retroactive ทั้ง Q

## Decision Record
- **Timing**: Retroactive — ไม่ว่า transfer เกิดเดือนไหนใน Q ก็หักฐานตั้งแต่ต้น
- **Scope**: เฉพาะ outlet ที่อยู่ใน baseMap (core cohort Mar) เท่านั้น
  - transfer_in / handover / new_sales / comeback ที่ย้ายออกไปใน Q → ไม่หักฐาน
- **Symmetric rule**: transfer_in ไม่เพิ่มฐาน (ไม่อยู่ใน NRR numerator อยู่แล้ว)

## Files Changed
- `src/07c_qnrr_view.js` — _qnrrCompute + bar render + callout card + list filter
- `src/styles_qnrr.css` — tout card, ghost segment, adj tag, chip styles

## Logic Changes (_qnrrCompute)

### New fields in return object
```js
{
  base_norm_original:     number,  // ฐานก่อนหัก transfer_out
  transfer_out_base_norm: number,  // จำนวนที่ถูกหัก (normalized ×30)
  transfer_out_outlets:   Array,   // [{outlet_id, gmv_norm, account_name, period_month}]
  base_norm:              number,  // ฐานที่ปรับแล้ว (= original − transfer_out)
}
```

### coreTransferOutSet scan
```js
// scan ทุก scoped rows ทั้ง Q หา transfer_out ที่อยู่ใน baseMap
scopedRows.forEach(r => {
  if (r.movement_type === 'transfer_out' && baseMap[r.outlet_id]) {
    coreTransferOutSet[r.outlet_id] = { gmv_norm, account_name, period_month }
  }
})
```

### NRR formula
```
NRR = nrr_curr_norm / adjusted_base_norm  ← ใช้ base_norm ที่หักแล้ว
```
