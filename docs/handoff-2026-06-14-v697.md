## Known Issues ที่ยังค้าง (สำคัญมาก)

### 🔴 Teamview card แรกโดนบัง (mobile)

**Root cause เข้าใจแล้ว แต่แก้ไม่สำเร็จ:**

Debug data จาก iOS Safari Web Inspector:
```
hdr_bcr_height: 370
lst_paddingTop: "486px"     ← JS set ถูก
computed_paddingTop: "0px"  ← CSS override!
card1_top: 374              ← card อยู่ผิดที่
```

**ปัญหาจริง:** `src/styles_teamview.css` line 48 (media query ≤430px):
```css
#teamview-content { padding: 0 16px calc(150px + env(safe-area-inset-bottom)) !important; }
```
shorthand `padding` + `!important` → override JS `element.style.paddingTop` ทุกครั้ง computed = 0px เสมอ

**วิธีแก้ที่ถูก (ยังไม่ implement):**
แก้ CSS rule ให้ใช้ CSS variable แทน:
```css
#teamview-content {
  padding-left: 16px !important;
  padding-right: 16px !important;
  padding-bottom: calc(150px + env(safe-area-inset-bottom)) !important;
  padding-top: var(--tv-content-offset, 0px) !important;
}
```
แล้วให้ `_tvSyncListOffset()` set CSS variable แทน inline style:
```js
document.documentElement.style.setProperty('--tv-content-offset', pt + 'px');
```

CSS variable + `!important` ชนะ regular CSS rule แต่ถูก override ได้โดย JS `setProperty`

**⚠️ ข้อควรระวัง:** v213d/e/f patches ใน `06_portview_teamview.js` อาจมี CSS อื่นใน `styles_teamview.css` ที่กระทบด้วย ต้อง grep ให้ครบก่อนแก้

## Key Architecture Notes

### Teamview CSS hierarchy (สำคัญมากสำหรับ next session)

```
styles_teamview.css  ← มี media query ≤430px ทับ padding!important
styles_portview.css  ← มี media query ≤430px ทับ padding!important ด้วย
shell.html           ← #teamview-content มี inline style padding
```

**อย่าแก้ด้วย inline style `element.style.paddingTop` เพราะ CSS `!important` ชนะ inline style ใน media query context**

ต้องใช้ CSS variable approach เท่านั้น

### Admin teamview cold boot path

```
doFade() → _splashPreFade() → showScreen('teamview')
→ allCriticalReady()=false (MAX_SHOW timeout)
→ post-fade block ไม่ fire
→ v689 fix: check scr-teamview.on + portviewBulkData.length > 0 → renderTeamview() directly
```

### Portview collapse state (v684)

```
window._pvCollapseIntent = true   ← user scroll collapsed
window._pvCollapseIntent = false  ← fresh navigation only
```
อย่า reset ใน filter/sort/search

### RenderBus signal path สำหรับ TL/Admin

portview CSV โหลดเสร็จ → signal `portview-for-teamview` (v688) → `_flush()` → `renderTeamview()`
แต่ถ้า splash ยัง active ตอน signal fire → signal หาย → v689 fix ใน post-fade block ช่วย
