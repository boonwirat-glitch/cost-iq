### 1. Echo Save Bug — Fixed (v728–v731)

**Root cause:** user กดบันทึก (cancel) หลัง live result → pipeline ยังรันอยู่ใน background → user กด Echo อีกครั้ง → `open()` reset `_sessionId = null` → analysis save ไม่รู้จะ update row ไหน → fallback insert สร้าง row ใหม่ไม่มีชื่อร้าน

**Fixes:**
- v728: ลบ duplicate `_checkRecoverBuffer` ออกจาก `open()`, เพิ่ม `_sessionId` guard
- v729: แก้ `open()` guard — evaluate phase ก่อน reset (แต่ยังไม่ถูกจุด)
- v730: เพิ่ม `processing` guard ใน `echoOpen()` (ไม่ใช่ตัวปัญหา)
- **v731 (fix จริง):** `open()` preserve `_sessionId` ถ้ามีค่าอยู่ไม่ว่า `_phase` จะเป็นอะไร, pipeline null `_sessionId` เองหลัง save เสร็จ

**Status:** ✅ ทดสอบแล้ว บันทึกถูกต้อง
