## Working Rules (ทุก session)

- **Snapshot ก่อนทุก significant change** — ชื่อ pattern: `snapshot/pre-{description}-vN`
- **Fetch ไฟล์สดก่อนแก้เสมอ** via GitHub blob API — ห้ามใช้ `raw.githubusercontent.com` (CDN cache) · ห้าม trust local state ข้าม session
- **Diagnose ก่อน fix** — อ่านโค้ดจริงก่อนเสนอ fix ห้าม trust "known issue" notes โดยไม่ verify (บทเรียน: kamEmail stale, Echo×Skills bridge stale ทั้งคู่)
- **`node --check` ทุก JS ก่อน push** — atomic commit ผ่าน Git Tree API: blobs → tree → commit → PATCH ref
- **Design-first** — mockup → อนุมัติ → build เสมอ ไม่ build UX/UI ใหม่โดยไม่มี direction ที่ confirmed ก่อน
