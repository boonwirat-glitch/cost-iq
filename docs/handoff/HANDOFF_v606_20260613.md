## Root Causes ที่แก้ Session นี้

### 1. Topbar ซ่อนหลัง Dynamic Island (v602)
Echo inject `_CSS` เข้า `<head>` ครั้งแรกแล้วไม่ remove ตอน `_unmount()` (by design) ทำให้ global selector `.topbar { padding:16px }` ค้างทับ `.topbar { padding-top: env(safe-area-inset-top) }` ของ app หลักตลอด session

### 2. Blank white screen (v604)
Race condition: `_unmount()` remove sheet ที่ 400ms (รอ slide-out animation 380ms) แต่ `_mount()` fire ที่ 50ms → เจอ `ci-fullsheet` เก่ายังอยู่ → guard `if (getElementById('ci-fullsheet')) return` → no-op → sheet เก่าไม่มี `ci-open` class อยู่ → `overflow:hidden` ค้าง → จอขาว

### 3. Double splash (v605)
`clients.claim()` ใน SW activate รับ control page ทันที → fire `controllerchange` → `location.reload()` ทุกครั้ง ไม่เฉพาะตอน user กด pill

## Architecture Notes ที่เปลี่ยน

**Echo CSS scoping:** ทุก selector ใน `_CSS` block (ยกเว้น `:root` tokens และ `body.echo-active`) ต้อง prefix ด้วย `#ci-fullsheet` เสมอ — ห้ามเพิ่ม global selector ใหม่โดยไม่ scope

**SW update flow:**
- `skipWaiting()` อยู่ใน install ไม่ได้ (removed v556)
- user กด pill → `window._senseUserRequestedUpdate = true` → `postMessage('SKIP_WAITING')` → SW activate → `clients.claim()` → `controllerchange` → ตรวจ flag → `location.reload()`
- SW activate เอง (tab reopen / clients.claim) → `controllerchange` fire แต่ไม่ reload เพราะไม่มี flag

**Echo tab structure (v606):**
- 3 tabs: บทสนทนา / ทักษะ / ลูกค้า
- `_overviewPanel(transcriptSummary, toneSignals)` = บทสนทนา
- `_skillsPanel(skillData)` = ทักษะ (ไม่มี PIPC bar, มี session_summary)
- `_customerPanel(intelData)` = ลูกค้า
- `_transcriptPanel()` ยังอยู่ใน return object แต่ไม่ถูกเรียก (backward compat)
- Live result และ session detail ใช้ renderer เดียวกัน
