## สิ่งที่ทำใน session นี้

### Goal
แก้ Sales mode boot → จอขาว ต้อง hard refresh ถึงจะขึ้น

### Root Cause (สรุปสุดท้าย)

ปัญหาซ้อนกัน 3 ชั้น ทั้งหมดมาจาก timing race ระหว่าง role detection กับ R2 fetch:

**ชั้นที่ 1 — `R2_FILES` เป็น frozen object**
`shell.html` ใช้ `Object.freeze()` ทุกชั้น ทำให้ `_patchR2FilesForSales()` ที่พยายาม assign
`R2_FILES['portview'] = 'sales_portview.csv'` เป็น silent no-op ทุกครั้ง
→ Sales โหลด KAM CSV, handover ยังอยู่ใน FOREGROUND → 404

**ชั้นที่ 2 — `DataRegistry._checkTier(1)` ใช้ `body.sales-mode`**
class นี้ถูก set ใน `_splashPreFade` (หลัง splash fade เท่านั้น) ไม่ใช่ตอน boot
→ `_isSales()` ใน DataRegistry คืน false ตอน markLoaded fire → รวม handover ใน Tier 1
→ handover 404 → `_tierReady[1]` ไม่เคยถูก set → `_flush HELD` ตลอด session

**ชั้นที่ 3 — ไม่มี single source of truth สำหรับ "this is a Sales session"**
Sales role check กระจาย 7+ จุด แต่ละจุดใช้ signal ต่างกัน ไม่มีจุดไหนพร้อมก่อน handover fetch เกิด

### Architecture หลัง v746 (stable)

```
Boot sequence สำหรับ Sales:

1. profile set (cache hit หรือ loadUserProfile resolve)
   → normalizeCurrentUserProfileRole()
   → _detectAndMarkSalesSession()          ← NEW: single entry point
       → _patchR2FilesForSales()
           → _salesR2Override populated     ← authoritative Sales signal
           → _cloudLoadedTabs.add('handover')  ← gate bypass
           → DataRegistry.markLoaded('handover') ← Tier 1 unlock

2. loadFromCloudflareR2()
   → _isSalesMode = _salesR2Override.length > 0  ← true เสมอแล้ว
   → handover ไม่อยู่ใน FOREGROUND
   → portview + history โหลด → allCriticalReady() = true

3. _fetchCloudflareFile gate
   → portview + history loaded → _splashDataReady() fire
   → splash fade ปกติ (ไม่ timeout)

4. DataRegistry._checkTier(1)
   → delegate to allCriticalReady() ← ไม่ใช้ body.sales-mode อีกต่อไป
   → _tierReady[1] = true → callbacks fire

5. RenderBus._flush
   → allCriticalReady() = true → ผ่าน gate
   → body.sales-mode set (จาก _splashPreFade) → renderSalesPortview()
```

**Sales signal hierarchy (single source):**
```
_salesR2Override.length > 0     ← authoritative (set ก่อน fetch ทุกกรณี)
getCurrentRole() === 'sales'    ← fallback ใน allCriticalReady + bg ETag
body.sales-mode                 ← CSS/render เท่านั้น (ไม่ใช้สำหรับ data gate)
```

## Key Learnings จาก session นี้

1. **`Object.freeze()` ใน shell.html config** — ทุก assign ต่อ frozen object เป็น silent no-op ใน strict mode หรือ non-strict mode ก็ไม่ error — ต้องระวังเสมอเมื่อแก้ R2 file routing
2. **Single entry point for role detection** — `_detectAndMarkSalesSession()` เรียกหลัง `normalizeCurrentUserProfileRole()` ทุกจุด เป็น pattern ที่ควรทำสำหรับ role-dependent setup ทุกอย่าง
3. **DataRegistry Tier 1 ไม่ควร embed role logic** — Tier 1 ควร delegate ไป `allCriticalReady()` ที่เป็น single source of truth สำหรับ "data ready" check
4. **อย่า fix ทีละ path** — Sales boot ถูก patch 10+ version เพราะแก้ที่ปลาย แก้จริงต้องหา single point ที่ role รู้แน่นอนก่อน fetch ใดๆ
