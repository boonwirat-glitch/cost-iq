### Root cause analysis (สำคัญมาก — เรียนรู้ไว้)

ปัญหา teamview card โดนบังมี **3 ชั้นซ้อนกัน** ต้องแก้ทุกชั้นพร้อมกันถึงจะหาย:

#### ชั้น 1 — `styles_teamview.css` (v213c)
```css
/* เดิม — shorthand !important ชนะ JS inline style */
#teamview-content { padding: 0 16px calc(150px + ...) !important; }
```
`padding` shorthand + `!important` → override `element.style.paddingTop` ทุกครั้ง
**Fix v698:** แตก shorthand → `padding-left`, `padding-right`, `padding-bottom` แยกกัน ไม่มี `padding-top`

#### ชั้น 2 — `styles_commission.css` (v210b) ← **ตัวร้ายหลัก**
```css
/* ไม่มี scope เลย — ยิง global ทุก state */
#teamview-content { padding-top: 0 !important; }
```
rule เก่าสมัย v210b ตั้งใจ zero padding-top ตอน commission overlay แต่ไม่มี scope
commission cockpit เป็น `position:fixed` ไม่ได้ขึ้นกับ padding-top ของ content เลย
**Fix v700:** ลบออกทั้งหมด — JS inline style ทำหน้าที่นี้แทน

#### ชั้น 3 — `shell.html` inline style shorthand
```html
<!-- เดิม — shorthand inline style -->
<div id="teamview-content" style="padding:0 16px 100px"></div>
```
เมื่อ JS set `element.style.paddingTop = '112px'` browser reset shorthand ทำให้ `padding-top` กลับเป็น 0
**Fix v701:** แยก inline style → `padding-left:16px;padding-right:16px;padding-bottom:100px`

### `_tvSyncListOffset()` — วิธีที่ถูกต้อง (v700)

เปลี่ยนจาก `hdr.BCR.height` (วัด full header height รวม padding ล่าง → over-estimate)
เป็น **overlap method** เหมือน `_pvSyncListOffset` ของ portview:

```js
var overlap = Math.max(0, Math.round(hdrRect.bottom - lstRect.top));
var pt = overlap > 0 ? overlap + 'px' : '0px';
if (lst.style.paddingTop !== pt) lst.style.paddingTop = pt;
```

CSS variable approach (`--tv-content-offset` / `setProperty`) ทดลองแล้ว **ไม่ work** เพราะ
CSS variable ที่ set ผ่าน JS `setProperty` บน `:root` ไม่ถูก apply เข้า rule ใน `@media` query
ใน PWA standalone (confirmed via DevTools: computed paddingTop = "0px" แม้ variable = "370px")

### Fix TL/Admin โหลดเข้า portview ว่าง (v701)

**Root cause:** skip-splash path มี 2 step ที่ขัดกัน:
1. L955: `showScreen('teamview')` ← ถูก (role routing)
2. L983-988: `setTimeout(0, showScreen(_rs.screen='portview'))` ← override กลับ!

`_rs.screen` = saved screen จาก session ก่อน → ถ้า TL เคย visit portview ก่อน logout
→ session ใหม่ L955 set teamview แต่ setTimeout(0) ยิง portview ทับ

**Fix v701 (`01_core.js`):**
```js
// v700: TL/Admin ข้าม _targetScreen resume block ทั้งหมด
var _isTlAdmin = _skipRoleForResume==='tl'||_skipRoleForResume==='admin';
if(!_isTlAdmin && typeof showScreen==='function'){
  // restore saved screen for non-TL/Admin only
  ...
}
```

### CSS variable ทำไมไม่ work ใน PWA (บันทึกไว้)

Debug data จาก Safari Web Inspector ต่อ iPhone:
```
document.documentElement.style.getPropertyValue('--tv-content-offset') → "370px"  ✓ set แล้ว
getComputedStyle(lst).paddingTop → "0px"  ✗ ไม่ถูก apply!
```

สาเหตุ: rule `padding-top:var(--tv-content-offset,0px)!important` อยู่ใน `@media (max-width:430px)`
CSS variable inheritance ผ่าน media query ใน PWA standalone mode ทำงานไม่สมบูรณ์บน iOS Safari
→ **อย่าใช้ CSS variable สำหรับ layout-critical properties ใน media query ใน PWA**

### กฎสำคัญ: `#teamview-content` padding-top (อย่าลืม)

```
❌ อย่า set padding-top:anything ใน CSS ใดๆ สำหรับ #teamview-content
   (ไม่ว่าจะเป็น !important หรือเปล่า — ถ้า scope ไม่แน่นพอจะทับ JS)
✅ padding-top ของ #teamview-content เป็นสิทธิ์ของ _tvSyncListOffset() เท่านั้น
✅ inline style ต้องเป็น shorthand-free: padding-left/right/bottom แยก ไม่ใช่ padding shorthand
```

## Key Architecture Notes (อัพเดท)

### CSS specificity war — บทเรียน session นี้

ทุกครั้งที่ JS ต้องการ set padding/layout property ผ่าน inline style:
1. grep ทุก CSS file หา selector นั้น + property นั้น
2. ถ้ามี `!important` → inline style **แพ้** → ต้องลบ `!important` ออกจาก CSS ก่อน
3. ถ้า inline style เป็น shorthand → JS set individual property ทับไม่ได้ → แยก shorthand ก่อน

### Portview vs Teamview padding-top approach (confirmed identical)

```
portview-list:     _pvSyncListOffset() → overlap → lst.style.paddingTop (ไม่มี CSS padding-top)
teamview-content:  _tvSyncListOffset() → overlap → lst.style.paddingTop (ไม่มี CSS padding-top) ← v700+
```
