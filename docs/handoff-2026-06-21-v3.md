## สิ่งที่แก้ไขใน session 2026-06-21 (หลัง handoff นี้ถูกเขียน)

Logic ต่อไปนี้ถูก invalidate ทั้งหมด:

1. **base_gmv** — เดิม filter `commercial_owner IN ('KAM','PM','ADMIN')` → ผิด
   ที่ถูก: นับ GMV ทุก order ใน March ไม่ filter portfolio

2. **Label lock** — เดิม re-classify ใหม่ทุกเดือน → ผิด
   ที่ถูก: lock ตั้งแต่เดือนแรก ไม่เปลี่ยนตลอดไตรมาส ยกเว้น transfer

3. **Handover fallback** — เดิม default handover ถ้าไม่มี exp_date → ผิด
   ที่ถูก: ดู first_portfolio_order date แล้วตัดสิน

4. **Expansion GMV=0** — เดิม flip เป็น transfer_in → ผิด
   ที่ถูก: ยังเป็น expansion GMV=0 label lock

5. **Comeback** — เดิม check pre-Mar owner เป็น KAM คนนี้ → ผิดสำหรับ master level
   ที่ถูก: Freshket-wide ไม่สนว่า KAM คนไหนดูแล

ดู spec ที่ถูกต้องทั้งหมดที่ `docs/qnrr_master_movement_design_v3.md`
