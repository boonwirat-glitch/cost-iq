## สิ่งที่ทำสำเร็จ

### Architecture เปลี่ยนใหม่ทั้งหมด (Echo)

เปลี่ยนจาก single Gemini call → 3-step pipeline:

```
Audio
  → Groq Whisper large-v3 (transcript + timestamps)
  → Gemini 2.0 Flash (diarize: แยก speaker จาก text)
  → Gemini 2.0 Flash (summarize: summary + notes + tone)
  → Claude Sonnet (analyze: 14 skills + OCPB)
```

Root cause เดิม: Gemini รับ audio 40+ นาทีแล้วทำทุกอย่างใน 1 call → output tokens หมด → วิเคราะห์แค่ส่วนต้น + hallucinate

### Worker v3 (`worker/freshket-sense-ai-proxy.js`)
- `/transcript` — Groq Whisper → segments[] (2-step: Groq text + Gemini diarize)
- `/summarize` — segments[] → summary JSON + notes + tone
- `/analyze` — segments[] + summary → skills + OCPB
- Legacy `/analyze-audio` และ `/transcribe` ยังอยู่

### Supabase
- เพิ่ม `transcript JSONB` column ใน `ci_sessions`
- เพิ่ม `pipeline_stage TEXT DEFAULT 'pending'` column

### App (`09_conv_intel.js`)
- `_processBlob()` → pipeline 3 ขั้น แต่ละขั้น save IDB ก่อน proceed
- `_callTranscript()` / `_callSummarize()` / `_callAnalyze()` — API calls แยก
- `_saveToSupabasePipeline()` — save transcript column ด้วย
- IDB version 1→2 เพิ่ม `pipeline` store สำหรับ recovery resume
- `_recoverBuffer()` — resume จาก transcript ที่มีอยู่แล้วได้ ไม่ต้อง re-transcript
- Progressive rendering — เข้าหน้า result ทันทีหลัง transcript เสร็จ Tab 2-3 update เองเมื่อเสร็จ
- `_updatePanel(idx)` helper — update single tab โดยไม่ re-render ทั้งหมด
- Tab บทสนทนา แสดง summary notes format (Google Meet style) แทน segments ยาวๆ
- เอา pain/med/opportunity pills ออกจาก OCPB facts render
- `echoOpen()` เพิ่ม guard — Sales bypass `allCriticalReady()`, KAM ต้องรอ Tier 1

## ค้างอยู่ / Known Issues

### Blank screen เมื่อ KAM กด Echo
ยังไม่ได้ root cause จริงครับ สิ่งที่รู้:
- เกิดเฉพาะตอน "เปิด app แล้วกด Echo" บน KAM role
- console ไม่แสดง CI pipeline log ใดๆ — แปลว่า mount ล้มเหลวก่อน log
- ไม่ใช่ SW cache เก่า (test หลัง clear cache แล้วยังเกิด)
- ต้องดู console จากตอนที่กด Echo จริงๆ ไม่ใช่ background logs

**จุดที่ต้องตรวจ:**
- `_buildHTML()` — ว่า inject HTML ครบไหม
- `_showScreen('ci-s-record')` — `.scr.on` CSS มีใน `_CSS` ไหม
- `_renderEchoState()` — condition ที่ทำให้ทุก element ถูก hide

### echoOpen guard
- Sales: bypass `allCriticalReady()` ใช้ `currentUserProfile.email` แทน — ยังไม่ได้ทดสอบจริง
- KAM: ใช้ `allCriticalReady()` — แต่ถ้า `_tierReady[1]` ไม่ถูก set หลัง MAX_SHOW timeout จะ block ตลอด

### Processing time
- Groq Whisper transcript audio 26 นาที → ใช้เวลา ~3 นาที → เกิน Cloudflare Pages→Worker timeout
- Audio ยาวมาก (>15 นาที) ยังมีปัญหา 524 timeout อยู่
- แนวทางแก้: chunking audio หรือ streaming response

### Summary notes format
- `/summarize` prompt สร้าง `notes[]` แล้ว แต่ยังไม่ได้ทดสอบว่า Gemini 2.0 Flash ทำได้ดีแค่ไหน
- `customer_said[]` ยังแสดงอยู่ใน Tab บทสนทนา อาจซ้ำกับ OCPB Tab ลูกค้า
