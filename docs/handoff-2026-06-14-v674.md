## สิ่งที่ทำใน session นี้

### Root Cause Analysis ก่อนลงมือ
- **Sales screens อยู่นอก `.main`** — `#scr-sales-portview` ฯลฯ อยู่ใน `swipe-grp-b` ไม่ได้อยู่ใน `.main{padding-top:var(--topbar-h)}` → ไม่ได้รับ topbar offset → content โผล่ใต้ topbar
- **Echo fullsheet padding-top fallback ผิด** — `env(safe-area-inset-top,44px)` = 44px fallback แต่ topbar จริง = `calc(54px + safe-area)` → บน Dynamic Island (safe-area≈59px) เนื้อหาโดนทับ
- **Pace=0% flash** — `renderPortviewTargetBar()` ยิงเมื่อ `portviewBulkData=[]` แต่ `target>0` จาก localStorage cache → `pct=0/target=0%`
- **IDB stale UI flash** — portviewBulkData จาก session เก่ายัง cache อยู่ใน IDB → render ด้วย design เก่าก่อน R2 fresh data มา
- **Collapse strip stale** — `_applyStrip()` guard `isCollapsed===collapsed` skip update แต่ strip className อาจ stale หลัง re-render
- **NRR 7500** — มาจาก `_commDefaultTiers` fallback เมื่อ `_tgtLoaded=false` แต่ CDS v565 มี tiers-ready gate แสดง shimmer แทนแล้ว → ไม่ใช่ bug ปัจจุบัน

### v673 — Sales + Echo + Pace fixes ✅
1. **`styles_sales.css`**: เพิ่ม `padding-top: var(--topbar-h)` + `padding-bottom` ให้ทุก `#scr-sales-*` และ `#scr-skills` ใน `body.sales-mode`
2. **`09_conv_intel.js`**: แก้ `#ci-fullsheet padding-top:env(safe-area-inset-top,44px)` → `env(safe-area-inset-top,0px)` ทั้ง 5 occurrences — ให้ sheet ใช้ safe-area จริงแทน 44px hardcode
3. **`07b_nrr_target.js`**: เพิ่ม guard ใน `renderPortviewTargetBar()` — ถ้า `portviewBulkData.length===0` return early + schedule self-heal retry
4. **SW bumped** to `sense-v673`

### v674 — KAM stale UI + Collapse fix ✅
5. **`01_core.js`**: เพิ่ม stale UI clear ใน `_showLoginOverlayClean()` — clear innerHTML ของ `portview-list`, `portview-summary-row`, `tgt-portview-bar`, `tgt-nrr-bar`, `pv-commission-strip`, `pv-compact-strip`, `portview-pace-bar` ทุกครั้ง logout → กัน IDB stale data flash
